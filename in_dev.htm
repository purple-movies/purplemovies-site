Below is a small score board example I put together to demonstrate my DBCommunicator class.

This class attempts to error-proof sending multiple SQL commands/queries to a server-side script. The idea here is to simplify sending/receiving data from Flash by allowing Flash to send SQL commands directly to the server. At which point my server-side script runs all input queries to update the database and then runs all output queries, returning the result in XML format.

I've written this so the server-side script obtains an array of input/output commands and so is able to update and return information from multiple tables and queries in one page request to the web-server. This example is using PHP server-side scripting, as it's the only language my web-server gives access to my database. I will port the code to ASP, ASP.NET and JSP if the need arises. This system is a work in progress and for more serious projects will require some security measures implemented.
